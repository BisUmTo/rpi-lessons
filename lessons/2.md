# Lezione 2: Raspberry Pi come Access Point Wi-Fi

In questa lezione vedremo come trasformare il Raspberry Pi in un router Wi-Fi (Access Point) per creare una rete locale e condividere la connessione internet dai dispositivi connessi via cavo.

## 1. Aggiornamento del Sistema Operativo
Ricollegarsi tramite SSH al Raspberry Pi come descritto nella Lezione 1.4.

Prima di iniziare, è fondamentale che il sistema operativo sia aggiornato. 

```bash
sudo apt update
sudo apt upgrade -y
```

### Troubleshooting
Se ci dovesse essere un errore *"The repository 'http://raspbian.raspberrypi.org/raspbian stretch Release' does no longer have a Release file."* modifichiamo il file `sources.list`:

```bash
sudo nano /etc/apt/sources.list
```

Commenta la riga esistente aggiungendo un # all'inizio e aggiungi la nuova riga, per ottenere quindi un file simile a questo:

```text
# deb http://raspbian.raspberrypi.org/raspbian/ stretch main contrib non-free rpi
deb http://legacy.raspbian.org/raspbian/ stretch main contrib non-free rpi
```

Per salvare il file quando si usa *nano* si deve premere `Ctrl+O` e poi `Ctrl+X`.

Se ci fosse un messaggio di avviso *"sudo: unable to resolve host gruppo3ITT0"* si può risolvere modificando il file `/etc/hosts` come descritto nella Lezione 1.6.

A questo punto, è possibile aggiornare il sistema operativo:

```bash
sudo apt update
sudo apt upgrade -y
```

## 2. Installazione del Software
Installa i pacchetti necessari per gestire l'Access Point e il server DHCP:

```bash
sudo apt install hostapd dnsmasq
```

Se viene richiesta la conferma, è sufficiente premere invio e automaticamente verrà scelta l'opzione scritta in maiuscolo.

## 3. Configurazione IP Statico (wlan0)
Perché il Raspberry Pi possa fare da router, la sua interfaccia Wi-Fi (`wlan0`) deve avere un indirizzo IP fisso.

1. Apri il file di configurazione di `dhcpcd`:
   ```bash
   sudo nano /etc/dhcpcd.conf
   ```
2. Aggiungi queste righe alla fine del file:
   ```text
   interface wlan0
   static ip_address=192.168.83.1/24
   nohook wpa_supplicant
   ```
3. Salva (`Ctrl+S`) ed esci (`Ctrl+X`).
4. Riavvia il servizio per applicare le modifiche:
   ```bash
   sudo service dhcpcd restart
   ```

## 4. Configurazione Server DHCP (dnsmasq)
Il servizio `dnsmasq` si occupa di assegnare automaticamente un indirizzo IP a ogni dispositivo (smartphone, PC, ecc.) che si collegherà alla tua rete Wi-Fi.

1. Rinomina il file di configurazione originale per sicurezza:
   ```bash
   sudo mv /etc/dnsmasq.conf /etc/dnsmasq.conf.orig
   ```
2. Crea un nuovo file di configurazione:
   ```bash
   sudo nano /etc/dnsmasq.conf
   ```
3. Inserisci il seguente contenuto:
   ```text
   interface=wlan0
   dhcp-range=192.168.83.2,192.168.83.254,255.255.255.0,24h
   ```
4. Salva ed esci, quindi avvia il servizio:
   ```bash
   sudo systemctl start dnsmasq
   ```

## 5. Configurazione Rete Wi-Fi (hostapd)
In questo passaggio definiremo il Nome della Rete (SSID) e la Password.

1. Crea il file di configurazione per l'Access Point:
   ```bash
   sudo nano /etc/hostapd/hostapd.conf
   ```
2. Incolla la seguente configurazione (devi personalizzare `ssid` e `wpa_passphrase`):
   ```text
   interface=wlan0
   driver=nl80211
   ssid=NomeTuaRete
   hw_mode=g
   channel=7
   wmm_enabled=0
   macaddr_acl=0
   auth_algs=1
   ignore_broadcast_ssid=0
   wpa=2
   wpa_passphrase=PasswordTuaRete
   wpa_key_mgmt=WPA-PSK
   wpa_pairwise=TKIP
   rsn_pairwise=CCMP
   country_code=IT
   ```

   Sostituisci `NomeTuaRete` con il nome che vuoi dare alla tua rete Wi-Fi (il nome del tuo gruppo es: `gruppo3ITT0`) e `PasswordTuaRete` con la password che vuoi usare per connetterti alla rete Wi-Fi.

3. Salva ed esci.

## 6. Abilitazione del Servizio hostapd
Dobbiamo indicare al sistema dove si trova il file di configurazione creato nel passaggio precedente.

1. Modifica il file di default:
   ```bash
   sudo nano /etc/default/hostapd
   ```
2. Cerca la riga `#DAEMON_CONF=""` e modificala (rimuovendo il `#`) in:
   ```text
   DAEMON_CONF="/etc/hostapd/hostapd.conf"
   ```
3. Sblocca il Wi-Fi, abilita e avvia il servizio:
   ```bash
   sudo rfkill unblock wifi
   sudo systemctl unmask hostapd
   sudo systemctl enable hostapd
   sudo systemctl start hostapd
   ```

## 7. Routing e NAT (Condivisione Internet)
Ora faremo in modo che il traffico dei dispositivi Wi-Fi venga "girato" (inoltrato) verso il cavo Ethernet per permettere la navigazione.

1. Abilita l'IP Forwarding modificando il file `sysctl`:
   ```bash
   sudo nano /etc/sysctl.d/99-sysctl.conf
   ```
2. Trova e decommenta (o aggiungi alla fine) la riga:
   ```text
   net.ipv4.ip_forward=1
   ```
3. Configura le regole del firewall (`iptables`) per permettere il NAT:
   ```bash
   sudo iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
   ```
4. Per rendere le regole permanenti anche dopo il riavvio, installa il pacchetto dedicato:
   ```bash
   sudo apt install iptables-persistent -y
   sudo netfilter-persistent save
   ```

## 8. Considerazioni Finali e Riavvio
Per essere sicuri che tutto funzioni correttamente (comprese le impostazioni regionali del Wi-Fi):

1. Esegui il tool di configurazione:
   ```bash
   sudo raspi-config
   ```
2. Naviga in **Localisation Options** -> **WLAN Country** e seleziona **IT Italy**.
3. Riavvia la Raspberry Pi:
   ```bash
   sudo reboot
   ```
4. Al riavvio, dovresti vedere la tua nuova rete Wi-Fi disponibile! In questo modo non necessiteremo più del cavo Ethernet per connetterci al Raspberry Pi.
